# 数据平台架构文档

## 概述

企业级数据仓库与数据质量平台，实现从多源异构数据摄入到业务指标交付的完整数据链路。

## 技术选型理由

| 技术 | 选型理由 |
|------|----------|
| SeaTunnel | 原生HTTP连接器（DataX无），Zeta引擎内置REST API，适合Airflow集成 |
| dbt | SQL优先的建模框架，自动血缘追踪（`{{ ref() }}`），内置测试框架 |
| Airflow | Python生态，成熟的DAG调度，丰富的Operator生态 |
| PostgreSQL | 免费开源，兼容dbt-postgres，支持双库隔离（元数据/数仓） |

## 数仓分层

| 层 | Schema | 物化 | 职责 |
|----|--------|------|------|
| ODS | ods | view | 源数据镜像，类型转换，附加`_ingested_at` |
| DWD | dwd | incremental | 去重、清洗、标准化、维度拆分 |
| DWS | dws | incremental | 按日/按类目聚合，产出服务层指标 |
| ADS | ads | table | 业务KPI、趋势排行、告警标记 |

## 数据血缘

1. dbt `ref()` 函数自动构建模型依赖DAG
2. `dbt docs generate` 生成可视化血缘图
3. 每张表注入 `source_system` + `_ingested_at` + `_layer` 审计列
